 修复排行榜查询性能问题                                                                                                 
 Context
                                     )
 排行榜页面（RankingView）加载企业排名时存在三个问题：
 1. 硬编码查询100家企业（RankingViewModel.cs:145），超过100家的企业会被遗漏
 2. N+1查询：对每个企业发起2次计数查询，100家企业产生201次数据库请求
 3. 全量拉取代替COUNT：计数方法先查出所有匹配记录再在内存里 .Count()，数据量大时极慢

 Approach

 采用内存分组聚合方案，不需要数据库改动，将排行榜加载从 2N+1 次查询优化到 3次查询。

 修改步骤

 1. Services/ICompanyService.cs — 添加无分页查询方法

 新增接口方法，返回所有活跃企业（排行榜排名需要全量）：
 Task<IEnumerable<Company>> GetAllActiveCompaniesAsync();

 2. Services/CompanyService.cs — 实现无分页查询

 实现 GetAllActiveCompaniesAsync()：
 - 查询条件：is_active = true
 - 不调用 .Range()，直接返回全部结果
 - 异常时返回空列表

 3. Services/IReviewService.cs — 添加批量计数方法

 新增两个接口方法，返回公司ID到数量的映射：
